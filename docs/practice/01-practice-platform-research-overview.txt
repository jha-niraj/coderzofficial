1. EXECUTIVE SUMMARY & VISION
=============================
The vision is to create an interactive "Practice" module (`/mock/practice`) that goes beyond traditional coding platforms like LeetCode or HackerRank. Instead of simply failing a user on hidden test cases or providing a final solution, the platform will act as an "AI Tutor" using the Socratic method. 

When a user attempts a problem, makes a mistake, and hits "Run/Submit", the AI will analyze their code/design and provide targeted hints, flowcharts, and step-by-step guidance to help them move from brute-force approaches to optimal solutions. Voice interaction will simulate a real-world pair-programming or mock interview scenario.

2. RESEARCH FINDINGS: WHAT THE MARKET IS DOING (2024-2025)
===========================================================
I researched modern platforms solving similar problems (Codemia, Thita.ai, LeetCode's recent AI features). Here are the key takeaways we can leverage:

A. Socratic Method & Hints over Solutions
- Research in AI coding education strongly favors the Socratic method over direct solution generation. Helping a user "think" rather than just giving them the code prevents knowledge gaps. 
- Platforms like Codecademy's new AI Assistant and Python Tutor use "subgoal hints"—nudging the user towards the next logical step without writing the code for them.

B. Deep Integration of Visuals (Flowcharts/Trees)
- Thita.ai and AlgoCademy heavily use visual pattern mastery. For DSA (especially recursion, trees, graphs), generating a visual representation of the execution flow helps users grasp *why* their code fails. Markdown-compatible flowchart libraries (like Mermaid.js) are perfect for this.

C. Iterative System Design (Codemia approach)
- Codemia operates like "LeetCode for System Design." They use interactive whiteboards where users drag-and-drop components, and the AI evaluates the architecture iteratively (scoring from 0-10) based on scalability, bottlenecks, and component choices.

D. Real-time Voice Feedback
- Voice AI for coding (like Serenade or standard LLM voice APIs) is changing the landscape. Instead of massive blocks of text, a quick voice clip saying, "You're on the right track with the hash map, but think about what happens when duplicate keys appear," keeps the user engaged.

General UI Layout (Inspired by "Learn" module):
-----------------------------------------------
[Left Sidebar] 
- Categories: DSA, System Design, Frontend, Backend.
- Subcategories: Arrays, Strings, Distributed Systems, APIs, etc.
- Responsive, collapsible list of questions.

[Main View - Split Screen]
- LEFT PANE: Problem description, constraints, examples.
- CENTER PANE: Active Workspace (Code Editor for DSA/Web, Excalidraw for SD).
- RIGHT PANE: AI Assistant / Mentor Chat. This area handles test case results, hints, Mermaid flowcharts, and voice transcripts.

4. USER FLOW & TRIGGER MECHANISM
================================

The AI should *not* evaluate every single keystroke (which is expensive and distracting).

1. User reads the problem.
2. User writes a solution / draws an architecture.
3. User hits "Run / Assess".
4. System executes the code against open test cases (or captures canvas state for SD).
5. If it fails, the trigger activates the AI Tutor.
6. The AI Tutor prompt is structured to:
   - Identify the user's current approach (e.g., "O(N^2) Brute Force").
   - Point out the exact line or logical flaw.
   - Generate a Mermaid/ASCII flowchart showing the flow.
   - Ask a guiding question (via Text + Voice) to lead them to the O(N) solution.
