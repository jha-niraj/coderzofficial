1. EXECUTIVE SUMMARY
================================================================================

The Learn module will serve as a structured, curriculum-grade learning platform.
Content is admin-curated (not user-generated) to ensure quality. The first
module is C++ Programming, structured as:

  MainCategory: Programming
  SubCategory: C++ (cpp)
  Topics: Individual Learns (30+ topics across 6 units)

Each Learn (topic) follows an interactive step flow:
  Explanation → Code → Quiz → Challenge → Resources → Summary

The detail page will feature:
  - Left: Step navigation sidebar
  - Center: Step content (markdown) with text-selection AI features
  - Right: Studio tab (notes + AI chat combined)

All content is seeded via a Prisma seed script for consistency and cost control.

9. PROGRESS TRACKING — COMPREHENSIVE MODEL
================================================================================

Current tracking (LearnProgress):
  ✅ currentStep, completedSteps[], progressPercent, isCompleted
  ✅ quizAnswers (JSON), challengeSubmissions (JSON)
  ✅ totalTimeSpent, lastAccessedAt

Additional tracking needed:

```
LearnProgress (extended):
  │
  ├── studioId          → Link to personal Study Notes
  ├── stepTimeSpent     → JSON { stepId: seconds }
  ├── aiInteractions    → Int (count of AI questions asked)
  ├── quizRetries       → JSON { stepId: retryCount }
  ├── bookmarkedSteps   → Int[] (step orders bookmarked)
  ├── notesPerStep      → JSON { stepId: noteContent } (quick notes)
  ├── lastStepViewedAt  → DateTime
  └── streakDays        → Int (consecutive days studied)
```

Progress tracking events to capture:
  1. Step viewed (stepId, timestamp, duration)
  2. Step completed (stepId, timestamp, method: quiz-pass/challenge-pass/manual)
  3. Quiz answered (stepId, questionId, answer, isCorrect, retryNumber)
  4. Challenge submitted (stepId, code, score, isCorrect)
  5. AI question asked (stepId, question, timestamp)
  6. Note saved (stepId, content)
  7. Mock interview completed (stepId, duration)
