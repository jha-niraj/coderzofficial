                    1. EXECUTIVE SUMMARY
================================================================================

PROBLEM:
--------
All three features (Studio, Space, Pathfinder) serve learning/tracking use cases
with overlapping capabilities. Users will struggle to answer:
  • "I want to learn React — where do I start?"
  • "I need to track my progress — which one?"
  • "I want to create content — Studio or Space?"

CORE FINDINGS:
--------------
• STUDIO = Personal notebook + AI on the right. Block editor (notes, quizzes,
  flashcards, code). No structured path, no progress tracking.

• SPACE = Collaborative learning journey. Timeline of steps (projects, studios,
  quizzes, links, videos). AI sidebar, progress tracking, members.

• PATHFINDER = Personal goal tracker. AI-generated daily tasks (quiz + coding),
  streaks, verification, groups. No notes, no collaboration.

OVERLAP:
--------
All three have: Quizzes, coding practice, AI content generation.
Studio + Space: Both have flashcards, block-like content.
Pathfinder + Space: Both have progress tracking, AI-generated content.

RECOMMENDATION:
---------------
KEEP: Pathfinder (personal goals, daily practice, verification)
KEEP: Spaces (collaborative courses, timeline, community)
MERGE: Studio → into Pathfinder & Spaces as an embedded component, not standalone

RATIONALE:
----------
1. Pathfinder = "I want to learn X" — clear, goal-driven, gamified.
2. Spaces = "Follow this course" or "Create a course" — collaborative, social.
3. Studio = "Take notes while learning" — should be a TOOL inside both, not a destination.

                    2. FEATURE #1: STUDIO
================================================================================

LOCATION: apps/main/app/(main)/studio

ROUTES:
-------
• /studio              — Dashboard: "Your Recent Studios"
• /studio/mystudios    — My Studios
• /studio/allstudios   — All Studios
• /studio/[slug]       — Studio Editor (notebook + AI panel)
• /studio/category/[slug] — Category browse

DESCRIPTION:
-----------
Studio is a personal AI-powered learning notebook. Notion-like block editor with
AI chat panel on the right. Users create studios to take notes, add quizzes,
flashcards, and code blocks.

KEY FILES:
----------
• studio/[slug]/page.tsx          — Loads StudioEditor
• components/studio/studio-editor.tsx — Main editor: blocks + AI panel
• components/studio/studio-block-editor.tsx — Block-based content
• components/studio/studio-ai-panel.tsx — Right-side AI chat
• _components/create-studio-sheet.tsx — Creation flow

CONTENT TYPES (Block Editor):
-----------------------------
• TEXT, HEADING, BULLET_LIST, NUMBERED_LIST
• CODE (with practice problems)
• QUIZ (AI-generated or manual)
• FLASHCARD (deck blocks)
• IMAGE, VIDEO, EMBED
• PRACTICE (code challenges)
• MOCK_INTERVIEW
• CALLOUT, DIVIDER

FEATURES:
---------
✓ Block-based rich editor
✓ AI chat panel (StudioAIPanel) — context-aware, persists chat history
✓ Quizzes, flashcards, code blocks inline
✓ Categories (PROGRAMMING, WEB_DEVELOPMENT, DSA, etc.)
✓ Visibility: PRIVATE, PUBLIC, COMMUNITY
✓ Auto-save (30s) + Ctrl+S
✓ Can link to Projects (projectId)
✓ Can be embedded IN Spaces (SpaceStepContentType.STUDIO)

DATABASE:
---------
Studio, StudioQuiz, StudioFlashcardDeck, StudioCodeBlock, StudioMediaBlock,
StudioChatMessage, StudioQuizAttempt, StudioFlashcardSession

WHAT STUDIO LACKS:
------------------
✗ No progress tracking
✗ No daily practice system
✗ No verification/assessment
✗ No collaboration
✗ No structured learning path
✗ Isolated — user must manually create everything

HOME PAGE PRESENCE:
------------------
• Continue Learning section shows recentStudios
• Links to /studio/[slug]

NAVIGATION:
-----------
Main sidebar: Studio (with children: Studio, My Studios, All Studios)

                    3. FEATURE #2: SPACE
================================================================================

LOCATION: apps/main/app/(main)/space

ROUTES:
-------
• /space              — Dashboard: My Spaces + Joined Spaces
• /space/myspaces     — My Spaces
• /space/allspaces    — All Spaces
• /space/[slug]       — Space detail: Timeline + Sidebar
• /space/[slug]/settings — Settings
• /space/[slug]/analytics — Analytics
• /space/category/[slug] — Category browse

DESCRIPTION:
-----------
Spaces is a collaborative learning platform. Creators build timeline-based
learning paths; learners join and follow steps. Think "course" or "classroom"
with real-time visibility of member progress.

KEY FILES:
----------
• space/[slug]/page.tsx — Two-column: SpaceTimeline + SpaceSidebar
• _components/space-timeline.tsx — Steps list/flow, AddContentSheet
• _components/space-sidebar.tsx — AI Chat, Quiz, Flashcards tabs
• _components/space-step-card.tsx — Step with complete/review
• _components/add-content-sheet.tsx — Add Project, Studio, Quiz, Flashcard, Link, Video

CONTENT TYPES (Space Steps):
----------------------------
• PROJECT (links to /projects)
• STUDIO (links to /studio)
• Learn (links to /Learns)
• QUIZ (AI-generated)
• FLASHCARD (AI-generated)
• LINK (external)
• VIDEO (YouTube)
• MOCK (coming soon)
• PDF, TEXT, AI_GENERATED

FEATURES:
---------
✓ Timeline/roadmap of sequential steps
✓ Flow view (canvas) + List view (traditional)
✓ Member system — join/leave, track progress
✓ Progress tracking per user
✓ AI Sidebar: Chat, Quiz, Flashcards (inline with timeline)
✓ Add content: Projects, Studios, Quizzes, Flashcards, Links, Videos
✓ Comments on steps (CommentsSheet)
✓ Branches (enableBranches for personalized paths)
✓ Assignment mode (due dates, teacher features)
✓ Visibility: PUBLIC, PROTECTED (access code), PRIVATE
✓ Analytics dashboard

SPACE SIDEBAR:
--------------
Three tabs: AI Chat | Quiz | Cards (Flashcards)
• AI Chat — context about space, generic chatWithAI
• Quiz — Click step from timeline → take quiz inline
• Cards — Click step from timeline → study flashcards inline

STEP COMPLETION:
----------------
• completeStep() marks step done
• Steps can link to Projects, Studio, Learns — user is redirected there
• Return URL: ?returnTo=space&spaceId=...&stepId=...

WHAT SPACE LACKS:
-----------------
✗ No personal goal setting
✗ No AI-generated daily plan (creator curates)
✗ No streaks / gamification
✗ No verification system

HOME PAGE PRESENCE:
------------------
None directly (no Space card on home)

NAVIGATION:
-----------
Main sidebar: Spaces (children: Spaces, My Spaces, All Spaces)

                    4. FEATURE #3: PATHFINDER
================================================================================

LOCATION: apps/main/app/(main)/(launchpads)/pathfinder

ROUTES:
-------
• /pathfinder              — Dashboard: Goals, Groups, Stats
• /pathfinder/[slug]      — Goal detail: Daily Practice View
• /pathfinder/[slug]/verify — Verification (quiz, coding, project, mock)
• /pathfinder/explore     — Browse public goals, copy to workspace
• /pathfinder/explore/[slug] — Goal preview

DESCRIPTION:
-----------
Pathfinder is a personal goal-based learning tracker. User sets a goal (e.g.
"Master Arrays and Strings in DSA"); AI generates daily sub-goals with quiz +
coding. Tracks progress, streaks, and offers verification.

KEY FILES:
----------
• pathfinder/page.tsx — PathfinderDashboard
• _components/pathfinder-dashboard.tsx — Goals list, groups, stats
• _components/create-goal-sheet.tsx — Goal creation (category, level, focus)
• [slug]/_components/daily-practice-view.tsx — Sub-goals list + Quiz/Coding
• [slug]/_components/subgoal-content-tabs.tsx — Resources, Quiz, Coding tabs
• [slug]/_components/subgoal-quiz.tsx — Quiz component
• [slug]/_components/subgoal-coding.tsx — Coding component
• [slug]/verify/_components/verification-page-client.tsx — Verification flow

GOAL CREATION:
--------------
• Title (what to learn)
• Category: DSA, WEB_DEVELOPMENT, FRONTEND, BACKEND, DEVOPS, AI_ML, etc.
• Level: BEGINNER, INTERMEDIATE, ADVANCED, EXPERT
• Duration: ONE_WEEK to SIX_MONTHS, CUSTOM
• Focus areas: Learns, practice, interview, projects, optimization, patterns
• Group (optional)
• Public/Private (private costs credits)

DAILY PRACTICE:
---------------
• Sub-goals (tasks) — user adds via CreateSubGoalSheet
• AI generates: Quiz questions, Coding problem, Resources (links, videos)
• Each sub-goal: Quiz tab + Coding tab (if hasCoding) + Resources
• Progress: completedSubGoals, quizScore, codingPassed
• Session stats: tasks done, quiz %, code solved

VERIFICATION:
-------------
• Quiz verification
• Coding verification
• Project verification
• Mock interview verification
• generateVerificationContent() — AI creates verification plan
• PathfinderVerification model stores status

ORGANIZATION:
-------------
• Groups (CreateGroupSheet) — color, emoji
• Assign goals to groups
• Ungrouped goals section

CREATOR ECONOMY:
----------------
• Public goals → others can copy (Explore page)
• CreatorEarningsSheet — earnings for popular goals
• PathfinderUsageWidget — usage stats

EXISTING LINK TO STUDIO:
------------------------
PathfinderGoal has studioId (nullable) — goals CAN link to a Studio.
Currently underused in UI.

WHAT PATHFINDER LACKS:
----------------------
✗ No note-taking (no block editor)
✗ No collaboration
✗ No course-like structure (creator-curated)
✗ No flashcards (only quiz + coding)

HOME PAGE PRESENCE:
------------------
• PathfinderGoalsCard — active goals, progress

NAVIGATION:
-----------
Main sidebar: Pathfinder (single item, no children)

Daily practice / streaks     |   ✗    |   ✗   |    ✓
Verification / assessment    |   ✗    |   ✗   |    ✓

CONFUSION SCENARIOS:
--------------------

1. "I want to learn React"
   → Studio? Create notes + quizzes.
   → Space? Join "Learn React" space.
   → Pathfinder? Create goal "Learn React Hooks".
   All three apply. User does not know which to pick.

2. "I want to practice coding"
   → Studio: Code blocks, practice blocks.
   → Space: Add project step, or quiz step.
   → Pathfinder: AI generates coding problems.
   Overlap.

3. "I want to create a course for others"
   → Space: Primary use case.
   → Pathfinder: Public goals can be copied.
   → Studio: Can be public, but no "course" structure.
   Space vs Pathfinder unclear for creators.

4. "I need to track my learning"
   → Pathfinder: Strong.
   → Space: Per-space progress.
   → Studio: None.

================================================================================
                6. USER JOURNEY COMPARISON
================================================================================

STUDIO JOURNEY:
---------------
1. User goes to /studio
2. Clicks "Create Studio"
3. Names it (e.g. "JavaScript Fundamentals")
4. Opens editor, adds blocks (text, quiz, flashcards, code)
5. Uses AI panel for help
6. Can share publicly or keep private
7. No completion tracking — user decides when "done"

SPACE JOURNEY (Learner):
------------------------
1. User goes to /space or discovers space
2. Joins a space (e.g. "Learn React from Scratch")
3. Sees timeline of steps
4. Completes steps (click → go to Project/Studio/Learn or take Quiz inline)
5. Marks steps complete
6. Uses AI sidebar for questions
7. Sees progress, other members

SPACE JOURNEY (Creator):
------------------------
1. User creates space
2. Adds content: projects, studios, quizzes, links, videos
3. Organizes timeline
4. Publishes (public/protected/private)
5. Others join and follow

PATHFINDER JOURNEY:
------------------
1. User goes to /pathfinder
2. Creates goal (e.g. "Master DSA Arrays")
3. AI generates plan (focus areas)
4. User adds daily sub-goals (or AI suggests)
5. Each sub-goal: take quiz, solve coding problem
6. Tracks: tasks done, quiz score, code solved, streak
7. Verifies: comprehensive quiz, coding, project, mock
8. Goal marked complete

================================================================================
                7. TECHNICAL INTEGRATION MAP
================================================================================

STUDIO ↔ SPACE:
---------------
• Space can add Studio as a step (SpaceStepContentType.STUDIO)
• addStudioToSpace() — adds studio to space timeline
• Clicking step redirects to /studio/[contentId]?returnTo=space&spaceId=...&stepId=...

STUDIO ↔ PATHFINDER:
--------------------
• PathfinderGoal.studioId exists in schema — underused
• No UI to "Open notes" or create studio from goal

SPACE ↔ PATHFINDER:
-------------------
• No direct integration
• Both can have quizzes, coding — different sources (Space: creator, Pathfinder: AI)

SHARED COMPONENTS:
------------------
• Quiz component (Quiz) — used in Space sidebar, Pathfinder SubGoalQuiz
• Flashcard — StudioFlashcardBlock used in Space sidebar, Pathfinder SubGoalContentTabs
• Code editor — used in Pathfinder SubGoalCoding
• chatWithAI — used in Space sidebar, Studio has StudioAIPanel (different impl)

================================================================================
                    8. RECOMMENDATIONS
================================================================================

OPTION A: KEEP ALL THREE (NOT RECOMMENDED)
------------------------------------------
Requires strong differentiation:
• Studio = "Personal notes only" — strip quizzes/coding, make it pure notebook
• Pathfinder = "Goals + practice"
• Spaces = "Courses"

Drawback: Major refactor to remove overlap; still confusing.

--------------------------------------------------------------------------------

OPTION B: MERGE STUDIO INTO PATHFINDER & SPACES (RECOMMENDED)
-------------------------------------------------------------

Convert Studio from standalone feature → component used inside Pathfinder & Spaces.

HOW:
----
1. Pathfinder goal → "Notes" tab/button
   • Creates or opens linked Studio (PathfinderGoal.studioId)
   • User takes notes while doing quizzes/coding
   • Context: notes are FOR this goal

2. Space step type STUDIO — already exists
   • Creator adds Studio as step
   • Learner clicks → opens Studio
   • Keep as-is

3. Remove standalone Studio from primary navigation
   • /studio becomes "My Studios" — list of studios linked to goals/spaces
   • Remove "Create Studio" from main nav
   • Create studio ONLY from: Pathfinder goal, Space add-content

BENEFITS:
---------
• Clear user decision: Pathfinder (personal) vs Spaces (collaborative)
• Notes become contextual, not a separate product
• No feature loss — all Studio functionality preserved
• Simpler mental model

--------------------------------------------------------------------------------

OPTION C: MERGE PATHFINDER INTO SPACES (ALTERNATIVE)
-----------------------------------------------------
Make Pathfinder = "Personal Space" — a space you create for yourself.
• Spaces: Personal | Collaborative
• AI generates content for personal spaces

DRAWBACKS:
• Spaces becomes very complex
• Different UX (personal vs collaborative)
• Pathfinder's daily practice + verification is unique

NOT RECOMMENDED.

--------------------------------------------------------------------------------

OPTION D: REMOVE ONE ENTIRELY
------------------------------
• Remove Studio: Lose note-taking. Users need notes. Bad.
• Remove Space: Lose collaborative courses. Bad.
• Remove Pathfinder: Lose goal tracking, verification, streaks. Bad.

NOT RECOMMENDED.

                    9. IMPLEMENTATION PLAN
================================================================================

IF IMPLEMENTING OPTION B (Merge Studio into Pathfinder & Spaces):

PHASE 1: Pathfinder + Studio (2–3 days)
---------------------------------------
1. In Pathfinder goal detail (DailyPracticeView), add "Notes" or "Open Notes" button
2. On click:
   • If goal.studioId exists → navigate to /studio/[studioSlug]
   • Else → call createStudio (link to goal), then navigate
3. When creating goal, optionally create linked studio
4. Update Pathfinder goal schema usage (studioId)

PHASE 2: Spaces + Studio (already done)
---------------------------------------
• Space add-content already has "Studio" option
• addStudioToSpace works
• No change needed

PHASE 3: Studio Access & Nav (1–2 days)
---------------------------------------
1. /studio page:
   • Rename to "My Studios" or "Studios"
   • Show studios with context: "Linked to: [Goal Name]" or "From Space: [Space Name]"
   • Remove "Create Studio" primary CTA
   • Add: "Studios are created when you add notes to a goal or add to a Space"

2. Navigation:
   • Remove "Studio" from main sidebar primary nav
   • OR move to secondary / tools section
   • Add "Notes" inside Pathfinder goal view

PHASE 4: Onboarding & Messaging (1 day)
---------------------------------------
1. Update onboarding:
   • "Set goals with Pathfinder — track progress, take notes"
   • "Join or create Spaces — learn with others"
2. Remove Studio as primary feature in marketing

PHASE 5: Continue Learning (optional)
-------------------------------------
• Home "Continue Learning" shows studios
• Consider: Only show studios linked to active goals/spaces
• Or: Show "Recent Studios" with context badge
