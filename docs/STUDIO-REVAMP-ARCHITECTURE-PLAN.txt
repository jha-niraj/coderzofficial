================================================================================
                    1. EXECUTIVE SUMMARY & PROBLEM STATEMENT
================================================================================

CURRENT SITUATION:
------------------
The existing Studio implementation is fundamentally flawed. It attempts to be
a Notion-like block editor with complex user interactions, but this approach
creates confusion and doesn't align with the platform's AI-first philosophy.

KEY PROBLEMS IDENTIFIED:

1. Users are given too much freedom without clear guidance
2. Block-based editing is complex and doesn't serve the learning purpose
3. No clear separation between AI-generated and user-written content
4. Auto-save every 30 seconds is inefficient and wasteful
5. Studio is not properly integrated with other modules (Pathfinder, Spaces)
6. Content structure doesn't support proper ordering and persistence

PROPOSED SOLUTION:
------------------
Transform Studio into an AI-FIRST learning workspace where:
- AI generates structured, markdown-based content
- Users interact primarily through AI prompts
- Content is automatically organized and saved based on user actions
- Clear separation between different content types (explanation, quiz, code, etc.)
- Seamless integration with Pathfinder goals and Space topics

CORE PHILOSOPHY:
----------------
"Studio is not a text editor. It's an AI-powered learning companion that
generates, organizes, and presents educational content in the most effective
format for each content type."

                    4. CONTENT BLOCK SYSTEM DESIGN
================================================================================

NEW CONTENT STEP MODEL:
-----------------------
Each piece of content is a "Step" with:
- Unique ID
- Order number (for sequencing)
- Type (explanation, quiz, code, etc.)
- Content data (varies by type)
- Metadata (created_at, updated_at, source)
- Status (draft, completed, archived)

STEP TYPES & THEIR STRUCTURE:
------------------------------

1. EXPLANATION (AI-Generated Markdown)
   ────────────────────────────────────
   {
     id: string
     order: number
     type: "explanation"
     content: string (markdown)
     source: "ai" | "user"
     metadata: {
       prompt: string
       model: string
       tokens_used: number
     }
   }

   RENDERING: React Markdown with syntax highlighting
   USER INTERACTION: Read-only, can regenerate with new prompt

2. USER NOTE (Rich Text)
   ──────────────────────
   {
     id: string
     order: number
     type: "note"
     content: string (HTML or Markdown)
     source: "user"
     metadata: {
       editor_type: "rich" | "markdown"
     }
   }

   RENDERING: Rich text editor (TipTap or similar)
   USER INTERACTION: Full editing capability
   SAVE TRIGGER: On blur or explicit save button

3. QUIZ (Interactive Assessment)
   ───────────────────────────────
   {
     id: string
     order: number
     type: "quiz"
     quiz_id: string (references StudioQuiz table)
     metadata: {
       topic: string
       difficulty: "easy" | "medium" | "hard"
       question_count: number
       generated_from: string (content context)
     }
   }

   RENDERING: Existing Quiz component
   USER INTERACTION: Take quiz, view results, retry
   SAVE TRIGGER: On quiz completion (results saved)

4. CODE PRACTICE (Coding Challenge)
   ──────────────────────────────────
   {
     id: string
     order: number
     type: "code"
     code_block_id: string (references StudioCodeBlock table)
     metadata: {
       language: string
       is_practice: boolean
       problem_title?: string
     }
   }

   RENDERING: Existing Code Block component (code-editor.tsx)
   USER INTERACTION: Write code, run, test
   SAVE TRIGGER: Debounced on code change (2 seconds)

5. IMAGE (AI-Generated or Uploaded)
   ──────────────────────────────────
   {
     id: string
     order: number
     type: "image"
     media_id: string (references StudioMediaBlock table)
     metadata: {
       source: "ai" | "upload"
       prompt?: string
       url: string
       width?: number
       height?: number
     }
   }

   RENDERING: Image with caption
   USER INTERACTION: View, download, regenerate (using falai api)
   SAVE TRIGGER: Immediate on generation/upload

6. VIDEO (YouTube Embed)
   ───────────────────────
   {
     id: string
     order: number
     type: "video"
     media_id: string
     metadata: {
       source: "youtube" | "vimeo"
       url: string
       title: string
       duration?: number
       generated_by: "xai" | "user"
     }
   }

   RENDERING: iframe embed
   USER INTERACTION: Watch video
   SAVE TRIGGER: Immediate on add

7. DOCUMENT (External Resource Link)
   ───────────────────────────────────
   {
     id: string
     order: number
     type: "document"
     metadata: {
       title: string
       url: string
       description?: string
       source: "xai" | "user"
       doc_type: "article" | "documentation" | "tutorial"
     }
   }

   RENDERING: Card with link preview
   USER INTERACTION: Click to open in new tab
   SAVE TRIGGER: Immediate on add

8. PROJECT (Minor/Major Project Suggestion)
   ──────────────────────────────────────────
   {
     id: string
     order: number
     type: "project"
     metadata: {
       project_type: "minor" | "major"
       title: string
       description: string
       difficulty: string
       estimated_time: string

   }

   USER INTERACTION:
     - Minor: Add to portfolio

   SAVE TRIGGER: Immediate on generation

9. MOCK INTERVIEW (Voice Interview Session)

     id: string

       topic: string

CONTEXT BUILDING FOR AI:
------------------------
When generating new content, include context from previous steps:

Example prompt structure:
```
SYSTEM: You are an educational content generator for a learning platform.

CONTEXT: The user is studying the following topic:
[Previous step 1: Explanation of JavaScript basics]
[Previous step 2: Quiz on variables and data types]
[Previous step 3: Code practice on functions]

USER REQUEST: {user_prompt}

TASK: Generate {content_type} that builds upon the previous content
and helps the user progress in their learning journey.
```

================================================================================
                    6. AUTO-SAVE MECHANISM
================================================================================

PROBLEM WITH CURRENT APPROACH:
------------------------------
Current: Auto-save every 30 seconds regardless of activity
Issues:
  - Wastes API calls when no changes made
  - Doesn't capture user intent properly
  - Can interrupt user during typing
  - No feedback on what was saved

NEW EVENT-DRIVEN AUTO-SAVE:
---------------------------

SAVE TRIGGERS:
--------------
1. User completes an action (not during typing)
2. User navigates away from studio
3. User explicitly clicks save
4. After AI content generation
5. After quiz completion
6. After code execution

IMPLEMENTATION STRATEGY:
------------------------

┌─────────────────────────────────────────────────────────────────────────┐
│                     AUTO-SAVE STATE MACHINE                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  STATE: IDLE                                                             │
│  ──────────                                                              │
│  - No pending changes                                                    │
│  - Last save timestamp displayed                                        │
│  - No save indicator shown                                              │
│                                                                          │
│  ▼ (User makes change)                                                  │
│                                                                          │
│  STATE: DIRTY                                                            │
│  ────────────                                                            │
│  - Changes pending                                                       │
│  - Show "Unsaved changes" indicator                                     │
│  - Start debounce timer (2 seconds)                                     │
│                                                                          │
│  ▼ (Timer expires OR explicit save)                                     │
│                                                                          │
│  STATE: SAVING                                                           │
│  ─────────────                                                           │
│  - Show "Saving..." indicator                                           │
│  - Disable further edits temporarily                                    │
│  - Send save request to API                                             │
│                                                                          │
│  ▼ (Save completes)                                                     │
│                                                                          │
│  STATE: SAVED                                                            │
│  ────────────                                                            │
│  - Show "Saved" checkmark (2 seconds)                                   │
│  - Update last save timestamp                                           │
│  - Return to IDLE state                                                 │
│                                                                          │
│  ▼ (Save fails)                                                         │
│                                                                          │
│  STATE: ERROR                                                            │
│  ────────────                                                            │
│  - Show error message                                                    │
│  - Keep changes in memory                                               │
│  - Offer retry button                                                   │
│  - Return to DIRTY state                                                │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘

SAVE GRANULARITY:
-----------------
Instead of saving entire studio content, save individual steps:

1. Step-level saves:
   - Only save the step that changed
   - Reduces payload size
   - Faster save operations
   - Better conflict resolution

2. Batch saves for related changes:
   - When AI generates content (step + metadata)
   - When user completes quiz (step + results)
   - When user finishes code (step + execution results)

SAVE API DESIGN:
----------------
POST /api/studios/:studioId/steps/:stepId/save - use the server actions please for the saving
Body: {
  content: {...},
  metadata: {...},
  order: number
}

Response: {
  success: boolean,
  step: {...},
  timestamp: string
}

OFFLINE SUPPORT (Future Enhancement):
--------------------------------------
- Store changes in IndexedDB
- Sync when connection restored
- Show offline indicator
- Queue saves for later

DETAILED SPECIFICATIONS FOR EACH CONTENT TYPE:

1. EXPLANATION (AI-Generated Markdown)
   ────────────────────────────────────

   USER FLOW:
   1. User selects "Explanation" from content type selector
   2. User enters prompt: "Explain React hooks in detail"
   3. System shows loading state
   4. AI generates comprehensive markdown content
   5. Content rendered with React Markdown
   6. User can regenerate with different prompt

   RENDERING REQUIREMENTS:
   - Syntax highlighting for code blocks (Prism.js)
   - Proper heading hierarchy
   - Lists, tables, blockquotes support
   - Math equations (KaTeX) if needed
   - Mermaid diagrams support

   AI PROMPT TEMPLATE:
   ```
   You are an expert educator. Create a comprehensive explanation
   of the following topic for a student learning programming.
   
   Topic: {user_prompt}
   
   Context from previous lessons:
   {previous_content_summary}
   
   Requirements:
   - Use clear, simple language
   - Include code examples where relevant
   - Use markdown formatting
   - Break down complex concepts
   - Length: 500-1000 words
   
   Format your response in markdown.
   ```

   SAVE BEHAVIOR:
   - Immediate save after generation
   - No user editing allowed (read-only)
   - Can regenerate to replace

2. USER NOTE (Rich Text Editor)
   ──────────────────────────────

   USER FLOW:
   1. User selects "Note" from content type selector
   2. Rich text editor appears
   3. User writes personal notes
   4. Auto-save on blur or after 2 seconds of inactivity
   5. User can format text (bold, italic, lists, etc.)

   EDITOR CHOICE: TipTap (React)
   - Lightweight and extensible
   - Markdown shortcuts support
   - Collaborative editing ready (future)
   - Good mobile support

   FORMATTING OPTIONS:
   - Headings (H1, H2, H3)
   - Bold, Italic, Underline
   - Bullet lists, Numbered lists
   - Code inline and blocks
   - Links
   - Blockquotes

   SAVE BEHAVIOR:
   - Debounced save (2 seconds after last keystroke)
   - Save on blur (when user clicks away)
   - Show save indicator

3. QUIZ (Interactive Assessment)
   ───────────────────────────────

   USER FLOW:
   1. User selects "Quiz" from content type selector
   2. User enters topic: "Test my knowledge on JavaScript closures"
   3. System analyzes previous content for context
   4. AI generates 5-10 questions
   5. Quiz rendered with existing Quiz component
   6. User takes quiz
   7. Results saved and displayed

   AI GENERATION:
   - Use previous content as context
   - Generate multiple choice questions
   - Include explanations for answers
   - Vary difficulty based on content

   QUIZ COMPONENT INTEGRATION:
   - Use existing Quiz component from components/main/quiz.tsx
   - Mode: "practice" (immediate feedback)
   - Show progress bar
   - Allow retry

   SAVE BEHAVIOR:
   - Save quiz structure on generation
   - Save user attempt on completion
   - Store score and time taken

4. CODE PRACTICE (Coding Challenge)
   ──────────────────────────────────

   USER FLOW:
   1. User selects "Code" from content type selector
   2. User enters prompt: "Create a coding challenge on array methods"
   3. AI generates:
      - Problem description
      - Starter code
      - Test cases
      - Solution (hidden)
   4. User writes code in editor
   5. User runs code to test
   6. Results displayed

   CODE EDITOR:
   - Use existing CodeEditor component
   - Monaco editor (VS Code engine)
   - Syntax highlighting
   - Auto-completion
   - Multiple language support

   EXECUTION:
   - JavaScript: We will send the code to the ai for the testing for now and later we will add the compiler for this one as well.
   - Other languages: Same as above for now using ai
   - Display console output
   - Show test case results

   SAVE BEHAVIOR:
   - Debounced save on code change (2 seconds)
   - Save execution results
   - Track attempts and solutions

5. IMAGE (AI-Generated Visual)
   ──────────────────────────────

   USER FLOW:
   1. User selects "Image" from content type selector
   2. User enters prompt: "A diagram showing React component lifecycle"
   3. System calls Fal.ai API
   4. Image generated and uploaded to Cloudinary
   5. Image displayed with caption

   IMAGE GENERATION:
   - Service: Fal.ai (Flux Pro model)
   - Already configured in codebase
   - Endpoint: /api/ai/generate-image

   CLOUDINARY UPLOAD:
   - Upload generated image
   - Get permanent URL
   - Store in StudioMediaBlock table

   DISPLAY:
   - Responsive image
   - Caption with prompt
   - Download button
   - Regenerate option

   SAVE BEHAVIOR:
   - Immediate save after generation
   - Store image URL and metadata

6. VIDEO (YouTube Resource)
   ───────────────────────────

   USER FLOW:
   1. User selects "Video" from content type selector
   2. Two options:
      a) AI-suggested: "Find videos about React hooks"
      b) Manual: Paste YouTube URL
   3. If AI: XAI searches and returns video suggestions
   4. User selects video
   5. Video embedded as iframe

   XAI INTEGRATION:
   - Endpoint: /api/ai/xai-resources

   VIDEO EMBED:
   - YouTube iframe
   - Responsive sizing
   - Title and description
   - Duration if available

   SAVE BEHAVIOR:
   - Immediate save on selection
   - Store video URL and metadata

7. DOCUMENT (External Resource)
   ───────────────────────────────

                    11. DATABASE SCHEMA CHANGES
================================================================================

NEW TABLES:
-----------

1. StudioStep
   ───────────
   Replaces the unstructured "content" JSON field

   CREATE TABLE studio_step (
     id VARCHAR(36) PRIMARY KEY,
     studio_id VARCHAR(36) NOT NULL,
     order_number INT NOT NULL,
     type ENUM(
       'explanation', 'note', 'quiz', 'code', 'image',
       'video', 'document', 'project', 'mock_interview', 'flashcard'
     ) NOT NULL,
     content TEXT,  -- For explanation, note
     metadata JSON,  -- Type-specific data
     source ENUM('ai', 'user') NOT NULL,
     status ENUM('draft', 'completed', 'archived') DEFAULT 'completed',
     created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
     updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
     
     FOREIGN KEY (studio_id) REFERENCES studio(id) ON DELETE CASCADE,
     INDEX idx_studio_order (studio_id, order_number),
     INDEX idx_type (type)
   );

2. StudioStepReference
   ────────────────────
   Links steps to their related entities (quiz, code block, etc.)

   CREATE TABLE studio_step_reference (
     id VARCHAR(36) PRIMARY KEY,
     step_id VARCHAR(36) NOT NULL,
     reference_type ENUM(
       'quiz', 'code_block', 'flashcard_deck', 'media_block',
       'project', 'mock_session'
     ) NOT NULL,
     reference_id VARCHAR(36) NOT NULL,
     
     FOREIGN KEY (step_id) REFERENCES studio_step(id) ON DELETE CASCADE,
     INDEX idx_step (step_id),
     INDEX idx_reference (reference_type, reference_id)
   );

3. StudioProjectSuggestion
   ────────────────────────
   Stores AI-generated project suggestions

   CREATE TABLE studio_project_suggestion (
     id VARCHAR(36) PRIMARY KEY,
     step_id VARCHAR(36) NOT NULL,
     project_type ENUM('minor', 'major') NOT NULL,
     title VARCHAR(255) NOT NULL,
     description TEXT NOT NULL,
     difficulty ENUM('beginner', 'intermediate', 'advanced') NOT NULL,
     estimated_hours INT,
     tech_stack JSON,  -- Array of technologies
     features JSON,  -- Array of feature descriptions
     created_project_id VARCHAR(36),  -- If user created the project
     added_to_portfolio BOOLEAN DEFAULT FALSE,
     
     FOREIGN KEY (step_id) REFERENCES studio_step(id) ON DELETE CASCADE,
     INDEX idx_step (step_id)
   );

4. StudioMockSession
   ──────────────────
   Stores mock interview session data

   CREATE TABLE studio_mock_session (
     id VARCHAR(36) PRIMARY KEY,
     step_id VARCHAR(36) NOT NULL,
     topic VARCHAR(255) NOT NULL,
     elevenlabs_conversation_id VARCHAR(255),
     transcript JSON,  -- Array of {role, content, timestamp}
     duration_seconds INT,
     ai_feedback TEXT,
     score INT,  -- 0-100
     status ENUM('pending', 'in_progress', 'completed') DEFAULT 'pending',
     started_at TIMESTAMP,
     completed_at TIMESTAMP,
     
     FOREIGN KEY (step_id) REFERENCES studio_step(id) ON DELETE CASCADE,
     INDEX idx_step (step_id)
   );
