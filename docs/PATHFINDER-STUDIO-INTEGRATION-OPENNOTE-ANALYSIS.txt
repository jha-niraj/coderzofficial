                    1. YOUR QUESTIONS ANSWERED
================================================================================

Q1: If we add Studio as a tab with notes + AI at bottom (user types "add
    content for how React works" → adds to notes), and Studio can do quizzes,
    coding, video, docs — why keep Resources, Videos, Docs, Quiz, Coding,
    Mock Practice tabs?

SHORT ANSWER:
------------
You should NOT keep all of them. Consolidate consumable content (Resources,
Videos, Docs, Flashcards) INTO the Studio tab as a unified "Learn" view.
Keep Quiz, Coding, and Mock as SEPARATE tabs because they require structured
completion tracking (score, pass/fail) that updates Pathfinder progress.

DETAILED REASONING:
------------------
• Resources, Videos, Docs, Flashcards = CONSUMABLE. User reads/watches. No
  completion state beyond "viewed". These can live inside Studio as sections
  or blocks.

• Quiz, Coding = ASSESSABLE. Must call submitSubGoalQuiz() and
  submitSubGoalCoding() to update quizCompleted, quizScore, codingPassed.
  Pathfinder dashboard and verification rely on these. Embedding in Studio
  is possible but requires block types with completion callbacks.

• Mock = REDIRECT. Goes to /mock/voice/interview/[sessionId]. Stays separate.

Q2: Should we integrate Studio as a component and remove the other tabs?

ANSWER:
-------
YES — but with a HYBRID approach:

1. Add "Notes" tab (Studio block editor) — user's personal notes + AI at bottom
2. MERGE Resources, Videos, Docs, Flashcards INTO the Notes tab as AI-
   generated sections (like Opennote's unified journal)
3. KEEP Quiz and Coding as separate tabs — cleaner completion tracking
4. KEEP Mock as separate tab — it's a redirect

Result: 4 tabs instead of 7
  • Notes (unified: AI content + user notes + AI prompt at bottom)
  • Quiz
  • Coding (if hasCoding)
  • Mock Practice

Q3: Where does the AI go? Right side vs bottom?

ANSWER:
-------
Move AI to BOTTOM of the Notes tab (as you suggested). User types a prompt
like "Explain how React useState works" → AI response is inserted as a new
block in the notes. This matches Opennote's "Add Instruction" pattern.

Current Studio: AI on right, chat-style. New: AI at bottom, "add to document"
behavior.

            2. CURRENT PATHFINDER TAB STRUCTURE (CODEBASE)
================================================================================

FILE: subgoal-content-tabs.tsx
TABS (in order):
-----------------
1. Resources   — content (markdown), codeExamples, dosDonts
2. Videos      — YouTube embeds from SubGoalResources.videos
3. Docs        — Links from SubGoalResources.documentations
4. Flashcards  — SubGoalResources.flashcards, StudioFlashcardBlock
5. Quiz        — SubGoalQuiz, aiQuizQuestions
6. Coding      — SubGoalCoding, aiCodingProblem (if hasCoding)
7. Mock        — Button to createPathfinderPracticeMockAndSession → redirect

DATA SOURCE: SubGoalResources (pathfinderStore, aiResources)
------------------------------
Generated when sub-goal is created (createSubGoal → generateSubGoalResources):
  • content: string (markdown)
  • codeExamples: { title, language, code, explanation }[]
  • dosDonts: { dos[], donts[] }
  • videos: { url, duration, description }[] (from Exa)
  • documentations: { url, type, description }[] (from Exa)
  • flashcards: { id, front, back, hint }[]

Quiz and Coding come from generateAIContentForSubGoal (separate from resources).

COMPLETION TRACKING:
--------------------
• Quiz: submitSubGoalQuiz() → updates PathfinderSubGoal.quizCompleted,
        quizScore. Used in daily-practice-view, verification.
• Coding: submitSubGoalCoding() → updates codingCompleted, codingPassed.
• Resources/Videos/Docs/Flashcards: No completion stored.

                3. OPENNOTE APPROACH — WHAT THEY DO
================================================================================

From Opennote (https://www.opennote.com) and the provided screenshot:

KEY PRINCIPLE: "Everything in one place"

FEATURES:
---------
• Notes — Rich editor, block-based
• AI Chat — "Ask Feynman" — knows context, explains
• Recorder — Audio/YouTube → notes, summaries

• Practice — Inline flashcards, quizzes (instant creation)

-----------------------------------

  1. Journal title
  2. "Add Instruction" — AI prompt box (e.g. "What makes Next.js a full-stack

  3. Pagination (1 of 3) — multi-step generation
  4. Diagram — e.g. "Socket IO Communication Flow" (visual)
  5. Quiz section — "Arrays in DSA practice" with multiple choice, "Check Answer"
  6. Detailed notes — "While Loops in Python" with syntax, examples, pitfalls

No separate tabs for Resources / Videos / Quiz. All in ONE flow.

AI BEHAVIOR: User adds "instruction" (prompt) → AI adds content to document.
             Not chat; content insertion.

================================================================================
                4. TWO INTEGRATION MODELS COMPARED
================================================================================

MODEL A: KEEP ALL TABS + ADD STUDIO
-----------------------------------
Tabs: Resources | Videos | Docs | Flashcards | Notes (Studio) | Quiz | Coding | Mock

Pros:
  • Minimal change
  • Clear separation

Cons:
  • 8 tabs — overwhelming
  • Redundancy: Resources has markdown + code examples; Studio could show same
  • User bounces between tabs to get full picture

--------------------------------------------------------------------------------

MODEL B: UNIFIED NOTES (OPENNOTE-STYLE) + ASSESSMENT TABS
---------------------------------------------------------
Tabs: Notes (unified) | Quiz | Coding | Mock

Notes tab contains:
  • AI-generated content (from SubGoalResources) as read-only sections:
    - Summary (content markdown)
    - Code Examples
    - Do's and Don'ts
    - Videos (embedded)
    - Documentation Links
    - Flashcards (inline)
  • User's editable notes (Studio block editor)
  • AI prompt at bottom: "Add explanation for X" → inserts block

Pros:
  • Like Opennote — one place to learn
  • Fewer tabs
  • Natural flow: read → take notes → practice (quiz/coding)

Cons:
  • Notes tab becomes long for content-heavy sub-goals
  • Need to design "add to document" AI (not just chat)

--------------------------------------------------------------------------------

MODEL C: FULL UNIFIED (QUIZ + CODING INLINE)
--------------------------------------------
Same as B, but Quiz and Coding also embedded as blocks in Notes.

Pros:
  • Maximum unification
  • True Opennote-style

Cons:
  • Quiz/Coding blocks need completion callbacks to update Pathfinder
  • Complex: block type "quiz" must call submitSubGoalQuiz on completion
  • Harder to retake quiz (need to reset block state)

WHY KEEP QUIZ AND CODING SEPARATE:
----------------------------------
1. Completion tracking is critical for Pathfinder (progress %, verification)
2. SubGoalQuiz and SubGoalCoding already handle this cleanly
3. Opennote may not have our verification/goal-tracking requirements
4. "Study" (Notes) vs "Test" (Quiz, Coding) is a clear mental model

WHY AI AT BOTTOM:
-----------------
• Frees horizontal space (no right panel)
• Matches "add instruction" pattern from Opennote
• User prompt → AI inserts block into "Your Notes" section

================================================================================
                8. IMPLEMENTATION IMPLICATIONS
================================================================================

PHASE 1: Create "Notes" Tab (Unified View)
------------------------------------------
1. New component: SubGoalNotesTab or PathfinderUnifiedNotes
2. Structure:
   - Section 1: AI content (from SubGoalResources) — content, codeExamples,
     dosDonts, videos, documentations, flashcards
   - Section 2: User notes (Studio block editor) — linked to PathfinderGoal
     studioId or create Studio on first edit
   - Section 3: AI prompt bar at bottom — "Add to notes" behavior

3. Remove tabs: Resources, Videos, Docs, Flashcards
4. Keep tabs: Quiz, Coding, Mock

PHASE 2: AI "Add to Notes" Behavior
-----------------------------------
Current Studio AI: sendChatMessage() — conversational.
New: sendChatMessage() with action "insert_block" or similar. Response is
     appended as a block to the user's notes section.

Options:
  a) Extend Studio AI: new endpoint "generateAndInsertBlock(studioId, prompt)"
  b) Pathfinder-specific: "addNoteBlock(subGoalId, prompt)" — creates/updates
     linked Studio, inserts block

PHASE 3: Link Studio to Sub-Goal
---------------------------------
PathfinderGoal has studioId. For sub-goal level, either:
  • One Studio per goal (all sub-goals share) — user's notes for whole goal
  • One Studio per sub-goal — more granular, more complex

Recommendation: One Studio per goal. Notes section in each sub-goal's Notes
tab shows:
  - AI content (sub-goal specific from SubGoalResources)
  - User notes from the goal's Studio — shared across all sub-goals
  - When user prompts "Add explanation for X", AI inserts into Studio;
    block can be tagged with subGoalId for optional filtering later.

Simpler: One Studio per goal. "Your notes" is the same across sub-goals.
User sees goal-level notes + sub-goal-specific AI content. This matches
"Learn React" — your notes accumulate for the whole goal, while each
sub-goal tab shows its own AI-generated learning material.

--------------------------------------------------------------------------------

OPENNOTE "ADD INSTRUCTION" PATTERN (Reference)
-----------------------------------------------
From the Opennote interface (https://www.opennote.com):

• Grey box at top: "What makes Next.js a full-stack framework other than
  just a React framework?" — this is the user's prompt (instruction)
• Pagination "1 of 3" — AI generates in steps/chunks
• Content appears BELOW the prompt in the same document
• Diagram, quiz, detailed notes — all flow in one scroll

For Pathfinder Notes tab:
• AI prompt bar at BOTTOM (your suggestion) — user types, clicks "Add"
• AI response inserted as new block in "Your Notes" section
• Same idea: prompt → content added to document, not side-by-side chat

PHASE 4: Tab Order and Default
-------------------------------
Default tab: Notes (was Resources).
Order: Notes | Quiz | Coding | Mock
