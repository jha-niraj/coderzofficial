1. EXECUTIVE SUMMARY
================================================================================

The Automatic Social Media Posting Integration enables CoderzLab users to 
share their verified learning achievements across multiple social platforms 
with a single click. The system uses AI to generate personalized, 
platform-optimized content that maintains the user's authentic voice.

CORE VALUE PROPOSITION:
• Eliminates friction in sharing achievements
• Generates professional, personalized posts
• Builds public learning portfolio automatically
• Increases accountability through public commitment
• Drives organic growth through user advocacy

SUPPORTED PLATFORMS:
• Twitter/X - Short-form achievement posts
• LinkedIn - Professional learning updates
• Future: Dev.to, GitHub Profile, Discord

KEY FEATURES:
• One-click posting to multiple platforms
• AI-generated personalized content
• User editing before posting
• Scheduled posting options
• Engagement analytics tracking

3.3 TOKEN STORAGE & SECURITY
─────────────────────────────

┌─────────────────────────────────────────────────────────────────────────────┐
│                      TOKEN SECURITY MODEL                                    │
└─────────────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────────────┐
│                           TOKEN VAULT                                       │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  STORAGE APPROACH:                                                          │
│  ─────────────────                                                          │
│  • Tokens encrypted at rest using AES-256-GCM                              │
│  • Encryption key stored in environment variables                          │
│  • Each user's tokens have unique IV (Initialization Vector)               │
│  • Tokens never logged or exposed in errors                                │
│                                                                             │
│  TOKEN STRUCTURE:                                                           │
│  ─────────────────                                                          │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  User Social Account                                                 │   │
│  │  ├── id: uuid                                                        │   │
│  │  ├── userId: user_uuid                                               │   │
│  │  ├── platform: "twitter" | "linkedin"                                │   │
│  │  ├── platformUserId: "123456789"                                     │   │
│  │  ├── platformUsername: "@johndoe"                                    │   │
│  │  ├── accessToken: encrypted_string                                   │   │
│  │  ├── refreshToken: encrypted_string                                  │   │
│  │  ├── expiresAt: datetime                                             │   │
│  │  ├── scope: ["tweet.write", "users.read"]                            │   │
│  │  ├── isActive: boolean                                               │   │
│  │  └── connectedAt: datetime                                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  REFRESH STRATEGY:                                                          │
│  ─────────────────                                                          │
│  • Check token expiry before each API call                                 │
│  • Refresh 5 minutes before expiry (buffer)                                │
│  • If refresh fails, mark connection as inactive                           │
│  • Notify user to re-authenticate                                          │
│                                                                             │
└────────────────────────────────────────────────────────────────────────────┘

5.2 PLATFORM-SPECIFIC POSTING
─────────────────────────────

TWITTER POSTING:
┌────────────────────────────────────────────────────────────────────────────┐
│  POST TO TWITTER                                                            │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. Retrieve user's Twitter access token                                    │
│  2. Validate token (refresh if needed)                                      │
│  3. Construct tweet payload:                                                │
│     {                                                                       │
│       "text": "Generated tweet content...",                                 │
│       "reply_settings": "everyone"                                          │
│     }                                                                       │
│  4. POST to https://api.twitter.com/2/tweets                               │
│  5. Parse response for tweet_id                                             │
│  6. Store tweet_id and URL                                                  │
│                                                                             │
│  ERROR HANDLING:                                                            │
│  • 401: Token expired → Refresh and retry                                  │
│  • 403: Permissions issue → Prompt re-auth                                 │
│  • 429: Rate limit → Queue for later                                       │
│  • 500: Server error → Retry with backoff                                  │
│                                                                             │
└────────────────────────────────────────────────────────────────────────────┘

LINKEDIN POSTING:
┌────────────────────────────────────────────────────────────────────────────┐
│  POST TO LINKEDIN                                                           │
├────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. Retrieve user's LinkedIn access token                                   │
│  2. Get user's LinkedIn URN (person URN)                                    │
│  3. Construct share payload:                                                │
│     {                                                                       │
│       "author": "urn:li:person:{member_id}",                               │
│       "lifecycleState": "PUBLISHED",                                        │
│       "specificContent": {                                                  │
│         "com.linkedin.ugc.ShareContent": {                                  │
│           "shareCommentary": { "text": "..." },                             │
│           "shareMediaCategory": "NONE"                                      │
│         }                                                                   │
│       },                                                                    │
│       "visibility": {                                                       │
│         "com.linkedin.ugc.MemberNetworkVisibility": "PUBLIC"               │
│       }                                                                     │
│     }                                                                       │
│  4. POST to https://api.linkedin.com/v2/ugcPosts                           │
│  5. Parse response for activity URN                                         │
│  6. Store activity URN and construct URL                                    │
│                                                                             │
└────────────────────────────────────────────────────────────────────────────┘
