1. CURRENT ARCHITECTURE RECAP
================================================================================

Models:
  Learn ──┬── LearnStep (order, type, content:TEXT, stepData:JSON)
          │     └── LearnCodeBlock (language, code, explanation)
          ├── LearnProgress (per user, completedSteps[], quizAnswers:JSON)
          ├── LearnLike, LearnBookmark, LearnComment, LearnView
          └── LearnLeaderboard

Categories:
  LearnMainCategory → LearnSubCategory → LearnTopic
  (LearnTopic is in the schema but NEVER USED in any action or UI)

Step Types (LearnStepType enum):
  EXPLANATION, CODE, VISUALIZATION, COMPARISON, QUIZ, CHALLENGE,
  INTERACTIVE, SUMMARY, RESOURCE, VISUAL, MOCK_INTERVIEW, PROJECT

Content Storage:
  - step.content → Markdown (TEXT field)
  - step.stepData → JSON (quiz options, challenge data, comparison items)
  - step.codeBlocks → Separate model (LearnCodeBlock)

Current Flow:
  Admin creates Learn → Adds Steps (AI generates content) → Publishes
  User reads step-by-step → Quizzes/Challenges tracked in LearnProgress

2. PROBLEM ANALYSIS — What's Wrong Today
================================================================================

PROBLEM 1: LearnTopic Model Does Nothing
─────────────────────────────────────────
The LearnTopic model exists in the schema with a relation to Learn, but:
  - No server actions use it (no CRUD, no filtering)
  - No UI references it
  - The Learn model has `topicId` but it's never set
  - Topics should organize Learns within a SubCategory (e.g., "React Hooks"
    under "React") but they're completely orphaned

  VERDICT: Either USE IT properly or REMOVE IT.
  RECOMMENDATION: Keep it, but actually use it for organizing learns
  within subcategories and for the navigation sidebar.

PROBLEM 2: Quiz Data Stored as JSON Blob in stepData
─────────────────────────────────────────────────────
Currently quiz data is stored as:
  stepData: {
    questions: [
      { question: "...", options: [{id, text, isCorrect}], explanation: "..." }
    ]
  }
  OR legacy format:
  stepData: { question: "...", options: [...], explanation: "..." }

Issues:
  - Two different formats (legacy single vs batch) → code handles both
  - Quiz answers tracked in LearnProgress.quizAnswers as JSON → hard to
    query, aggregate, or build analytics
  - No way to track individual question performance across users
  - No retry tracking per question (just per step)
  - Quiz questions can't be shared/reused across steps
  - No difficulty level per question
  - No support for code-based questions (e.g., "What does this code output?")
  - Parsing JSON on every render is fragile

  VERDICT: Quiz needs its own model with structured fields.

PROBLEM 3: Challenge Data Also JSON Blob
─────────────────────────────────────────
Challenge (coding exercise) data stored as:
  stepData: {
    starterCode: "...",
    solution: "...",
    hints: ["..."],
    testCases: [{input, expectedOutput}],
    language: "javascript"
  }

And user submissions tracked in LearnProgress.challengeSubmissions as JSON:
  { stepId: { code, passed, submittedAt } }

Issues:
  - Test cases are JSON strings, not structured → can't run automated tests
  - Only stores pass/fail, not granular test results
  - No support for multiple languages per challenge
  - No versioning of submissions
  - Submission history is a flat JSON blob

  VERDICT: Challenges need better test case structure.

PROBLEM 4: Content Is One Giant Markdown Field
───────────────────────────────────────────────
Each step has a single `content: String @db.Text` field that stores
everything in Markdown. This means:
  - Can't search within specific content blocks
  - Can't reorder or restructure content sections
  - Heavy for rendering (1 massive Markdown block per step)
  - No way to have "expandable" sections, "key takeaways", "warnings"
    as structured data

  VERDICT: This is FINE for explanatory content. Markdown is the right
  format for educational text. But we should add structured companion
  fields for things like key takeaways and warnings.

PROBLEM 5: No Interview Questions / Flashcard System
─────────────────────────────────────────────────────
The user specifically wants:
  "Add another step which would be like interview questions, where we
   should add the questions that could be asked in an interview using
   the flash card, so the user can read that as well."

This doesn't exist at all. Need a new step type: INTERVIEW_QUESTIONS
with a flashcard UI.

PROBLEM 6: Quiz Module Can't Show Code
───────────────────────────────────────
The user said:
  "We do need to update the quiz module to be able to handle a code
   which will be shown using a code editor, which will be read only."

Currently the Quiz component (quiz.tsx) already has a `codeSnippet`
field in QuizQuestion interface, but it's not connected to the
stepData generation. The AI generates questions without code context.

  VERDICT: Need `codeSnippet` field in quiz generation and in the
  stepData schema so code-based questions are properly supported.

PROBLEM 7: Single Challenge Per Step
─────────────────────────────────────
Each CHALLENGE step can only have one challenge (one starterCode,
one solution). The user mentioned "there is only one challenge" —
this is limiting. Should support multiple progressive challenges.

3. SCHEMA REDESIGN — What Needs to Change
================================================================================

CHANGE 1: Add LearnQuizQuestion Model (extract from JSON)
──────────────────────────────────────────────────────────

```prisma
model LearnQuizQuestion {
    id        String   @id @default(cuid())

    stepId    String
    step      LearnStep @relation("StepQuizQuestions", fields: [stepId], references: [id], onDelete: Cascade)

    order     Int      @default(0)
    question  String   @db.Text
    type      QuizQuestionType @default(SINGLE_CHOICE)

    // Options stored as structured JSON (not markdown)
    options   Json     // [{id: "a", text: "...", isCorrect: true/false}]

    // Explanation shown after answering
    explanation String? @db.Text

    // Optional code snippet (displayed in read-only CodeEditor)
    codeSnippet String? @db.Text
    codeLanguage String? @default("javascript")

    // Difficulty for analytics
    difficulty  QuizDifficulty @default(MEDIUM)

    // Points awarded
    points    Int      @default(10)

    createdAt DateTime @default(now())

    // Track individual answers
    answers   LearnQuizAnswer[] @relation("QuizQuestionAnswers")

    @@index([stepId])
    @@index([order])
    @@map("LearnQuizQuestion")
}

enum QuizQuestionType {
    SINGLE_CHOICE    // One correct answer
    MULTIPLE_CHOICE  // Multiple correct answers
    CODE_OUTPUT      // "What does this code output?" with code snippet
    TRUE_FALSE       // True/False question
}

enum QuizDifficulty {
    EASY
    MEDIUM
    HARD
}

model LearnQuizAnswer {
    id          String   @id @default(cuid())

    questionId  String
    question    LearnQuizQuestion @relation("QuizQuestionAnswers", fields: [questionId], references: [id], onDelete: Cascade)

    userId      String
    user        User     @relation("LearnQuizAnswers", fields: [userId], references: [id], onDelete: Cascade)

    selectedOptions String[]  // Array of option IDs selected
    isCorrect   Boolean
    attemptNumber Int     @default(1)
    timeTaken   Int?     // seconds to answer

    answeredAt  DateTime @default(now())

    @@index([questionId])
    @@index([userId])
    @@index([questionId, userId])
    @@map("LearnQuizAnswer")
}
```

WHY: This gives us:
  ✅ Per-question analytics (which questions are hard?)
  ✅ Per-user answer history with retry tracking
  ✅ Code snippets in questions natively
  ✅ Multiple question types
  ✅ Difficulty levels for adaptive learning
  ✅ Points system for gamification

CHANGE 2: Add LearnFlashcard Model (Interview Questions)
────────────────────────────────────────────────────────

```prisma
model LearnFlashcard {
    id        String   @id @default(cuid())

    stepId    String
    step      LearnStep @relation("StepFlashcards", fields: [stepId], references: [id], onDelete: Cascade)

    order     Int      @default(0)
    category  String?  // e.g., "Conceptual", "Coding", "Behavioral", "Tricky"

    // Front of the card (the question)
    question  String   @db.Text

    // Back of the card (the answer)
    answer    String   @db.Text

    // Optional code snippet on the question side
    codeSnippet  String? @db.Text
    codeLanguage String? @default("javascript")

    // Difficulty tag
    difficulty FlashcardDifficulty @default(MEDIUM)

    // Tags for filtering
    tags      String[]

    // Spaced repetition data (per user tracked in progress)
    createdAt DateTime @default(now())

    @@index([stepId])
    @@index([order])
    @@map("LearnFlashcard")
}

enum FlashcardDifficulty {
    EASY
    MEDIUM
    HARD
    EXPERT
}
```

4. QUIZ MODULE REDESIGN — Structured Data, Not Markdown
================================================================================

CURRENT FLOW (BAD):
  1. Admin clicks "Generate Quiz" in editor
  2. AI returns JSON: {questions: [{question, options, explanation}]}
  3. Stored in stepData JSON blob
  4. On render, parsed from JSON → passed to Quiz component
  5. User answers → stored in LearnProgress.quizAnswers as JSON blob
  6. No way to know which specific questions are hard across all users

NEW FLOW (GOOD):
  1. Admin clicks "Generate Quiz" in editor
  2. AI returns structured questions
  3. Each question saved as LearnQuizQuestion record in DB
  4. On render, questions fetched with proper includes
  5. User answers → saved as LearnQuizAnswer records
  6. Analytics: "Question X has 30% success rate" → improve content
  7. Code-based questions show read-only CodeEditor with the snippet

7. CONTENT STRUCTURE — Separation of Concerns
================================================================================

Current state: step.content (Markdown) + step.stepData (JSON) works
well enough. The real issue is that quiz and flashcard data should
be in proper DB models, NOT in stepData JSON.

RECOMMENDED DATA SPLIT:
┌──────────────────────────────────────────────────────┐
│ What stays in step.content (Markdown TEXT):           │
│  - Explanations, tutorials, theory                   │
│  - Code examples within markdown (```blocks)         │
│  - Diagrams via Mermaid syntax                       │
│  - Rich formatted educational content                │
├──────────────────────────────────────────────────────┤
│ What stays in step.stepData (JSON):                  │
│  - Comparison items [{title, pros, cons}]            │
│  - Challenge config {starterCode, solution, hints}   │
│  - Resource links [{url, title, type}]               │
│  - Visualization config                              │
│  - Summary keyTakeaways, assignment                  │
├──────────────────────────────────────────────────────┤
│ What moves to SEPARATE MODELS:                       │
│  - Quiz questions → LearnQuizQuestion model          │
│  - Interview flashcards → LearnFlashcard model       │
├──────────────────────────────────────────────────────┤
│ What stays in LearnCodeBlock model:                  │
│  - Standalone code examples with explanations        │
│  - Runnable code snippets                            │
│  - Highlighted line configurations                   │
└──────────────────────────────────────────────────────┘
