2.3 CURRENT AI GENERATION (project-generator.ts)
-------------------------------------------------

Current Generation Flow:
1. Create OpenAI thread
2. Send user input (title, description, stacks, etc.)
3. Run assistant with pre-configured assistant_id
4. Poll for completion (~70 seconds)
5. Parse JSON response
6. Save to database:
   - ProjectV2 (main record)
   - ProjectV2Page[] (basic pages)
   - ProjectV2Task[] (flat task list)
   - ProjectV2Quiz (if includeAssessment)
   - ProjectV2KnowledgeBase (if includeAssessment)

ISSUES WITH CURRENT GENERATION:
-------------------------------
❌ learningObjectives - Too generic, users skip these
❌ prerequisites - Usually obvious from tech stack
❌ coreFeatures vs advancedFeatures - Artificial distinction
❌ Pages lack depth - No route, layout, component specs
❌ Tasks are flat - No sprint grouping or dependencies
❌ No PRD document for project context
❌ No data architecture (models, APIs)
❌ No folder structure guidance
❌ No setup/environment guide
