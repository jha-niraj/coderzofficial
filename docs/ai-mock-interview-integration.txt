1. OVERVIEW & PURPOSE
================================================================================

The AI Mock Interview system bridges two platforms:

1. HIRING PLATFORM (apps/hiring/)
   - Used by HR/Companies
   - Configure interview processes
   - Add rounds with mock interview support
   - View candidate mock scores
   - Schedule real interviews

2. MAIN PLATFORM (apps/main/)
   - Used by Candidates/Students
   - Browse companies and jobs
   - Practice AI mock interviews
   - Track interview preparation progress
   - Apply to jobs

KEY GOAL: Enable candidates to practice company-specific interview rounds
         while HR gets visibility into candidate preparation levels.

2. CURRENT SYSTEM ARCHITECTURE
================================================================================

DATABASE SCHEMA (Shared by both platforms)
------------------------------------------

InterviewProcess (created by HR in Hiring Platform)
├── id
├── companyId
├── name
├── description
├── estimatedDurationWeeks
├── isDefault
└── rounds[] → InterviewRound

InterviewRound (configured per process)
├── id
├── processId
├── roundNumber
├── roundType (PHONE_SCREEN, TECHNICAL_CODING, SYSTEM_DESIGN, etc.)
├── title
├── durationMinutes
├── format (VOICE, VIDEO, LIVE_CODING, etc.)
├── description (candidate-facing)
├── hasMockInterview (boolean) ← KEY FLAG
├── mockKnowledgeBase (custom AI context)
└── mockSessions[] → JobMockSession

JobMockSession (candidate practice sessions)
├── id
├── userId (candidate)
├── jobId (optional - specific job)
├── companyId
├── roundId → InterviewRound
├── sessionType (VOICE, CODING, SYSTEM_DESIGN, BEHAVIORAL)
├── status (SCHEDULED, IN_PROGRESS, COMPLETED, etc.)
├── conversationId (ElevenLabs)
├── overallScore
├── categoryScores (JSON)
├── aiAnalysis (JSON)
└── timestamps

Job
├── interviewProcessId → InterviewProcess
└── (links job to company's interview process)

B. STARTING A MOCK SESSION
--------------------------

Server Action: apps/main/actions/companies/mock.ts

```typescript
export async function startCompanyMockSession(
    companySlug: string,
    roundId: string,
    jobId?: string
) {
    const session = await auth()
    if (!session?.user?.id) throw new Error("Unauthorized")
    
    // Get round details for session type
    const round = await prisma.interviewRound.findUnique({
        where: { id: roundId },
        include: { process: true }
    })
    
    // Determine session type from round type
    const sessionType = mapRoundTypeToSessionType(round.roundType)
    
    // Create mock session
    const mockSession = await prisma.jobMockSession.create({
        data: {
            userId: session.user.id,
            companyId: round.process.companyId,
            roundId: round.id,
            jobId: jobId || null,
            sessionType,
            status: "SCHEDULED"
        }
    })
    
    return mockSession
}
```


C. HR VIEWING CANDIDATE SCORES
------------------------------

Server Action: apps/hiring/actions/mock/index.ts

```typescript
export async function getMockSessionsOverview() {
    const member = await getCurrentCompanyMember()
    
    const sessions = await prisma.jobMockSession.findMany({
        where: { 
            companyId: member.companyId,
            status: "COMPLETED"
        },
        include: {
            user: { select: { name: true, email: true, image: true } },
            round: { select: { title: true, roundType: true } },
            job: { select: { title: true, slug: true } }
        },
        orderBy: { completedAt: "desc" }
    })
    
    return sessions
}
```
