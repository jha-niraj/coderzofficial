================================================================================
                    INTERVIEW ACCEPTANCE FLOW DOCUMENTATION
                           Coderz Hiring Platform
================================================================================

This document outlines the complete interview acceptance flow for the Coderz 
hiring platform, covering both the HIRING app (for HR/recruiters) and the 
MAIN app (for candidates).

================================================================================
                              OVERVIEW
================================================================================

The interview acceptance flow consists of the following stages:
1. Application Review (Hiring App)
2. Shortlisting (Hiring App)
3. Interview Scheduling (Hiring App)
4. Interview Notification (Main App - Candidate)
5. Interview Participation (Both Apps)
6. Interview Evaluation (Hiring App)
7. Final Decision (Hiring App)

================================================================================
                    DATA MODELS
================================================================================

Interview Model (Prisma Schema):
─────────────────────────────────────────────────────────────────────────────

model Interview {
  id            String      @id @default(cuid())
  applicationId String
  application   JobApplication @relation(fields: [applicationId], references: [id])
  
  scheduledAt   DateTime
  duration      Int         @default(60) // minutes
  
  meetingLink   String      @unique
  meetingType   InterviewType @default(VIDEO)
  
  status        InterviewStatus @default(SCHEDULED)
  
  notes         String?     // HR notes
  feedback      String?     // Post-interview feedback
  rating        Int?        // 1-5 rating
  
  interviewers  Interviewer[]
  
  createdAt     DateTime    @default(now())
  updatedAt     DateTime    @updatedAt
}

enum InterviewType {
  VIDEO
  PHONE
  IN_PERSON
  TECHNICAL
}

enum InterviewStatus {
  SCHEDULED
  CONFIRMED
  IN_PROGRESS
  COMPLETED
  CANCELLED
  NO_SHOW
}

HIRING APP SERVER ACTIONS:
─────────────────────────────────────────────────────────────────────────────

┌─────────────────────────────────────────────────────────────────────────────┐
│  scheduleInterview(applicationId, scheduledAt, notes?)                      │
│  ─────────────────────────────────────────────────────                      │
│  Input:                                                                     │
│    - applicationId: string                                                  │
│    - scheduledAt: Date                                                      │
│    - notes?: string                                                         │
│                                                                             │
│  Process:                                                                   │
│    1. Validate application exists and status allows scheduling              │
│    2. Generate unique meeting link                                          │
│    3. Create Interview record                                               │
│    4. Update JobApplication status to INTERVIEW_SCHEDULED                   │
│    5. Add activity log                                                      │
│    6. Trigger notification to candidate                                     │
│                                                                             │
│  Output:                                                                    │
│    - success: boolean                                                       │
│    - data: { interviewId, interviewLink }                                  │
│    - error?: string                                                         │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│  rejectApplication(applicationId, reason)                                   │
│  ────────────────────────────────────────                                   │
│  Input:                                                                     │
│    - applicationId: string                                                  │
│    - reason: string (markdown formatted)                                    │
│                                                                             │
│  Process:                                                                   │
│    1. Validate application exists                                           │
│    2. Update status to REJECTED                                             │
│    3. Store rejection reason                                                │
│    4. Add activity log                                                      │
│    5. Send rejection email to candidate                                     │
│                                                                             │
│  Output:                                                                    │
│    - success: boolean                                                       │
│    - error?: string                                                         │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│  makeMessageProfessional(message)                                           │
│  ────────────────────────────────                                           │
│  Input:                                                                     │
│    - message: string (raw message text)                                     │
│                                                                             │
│  Process:                                                                   │
│    1. Send message to OpenAI GPT-4                                          │
│    2. Prompt: "Transform this message into a professional, polite           │
│       rejection message while maintaining the key points..."                │
│    3. Return enhanced message                                               │
│                                                                             │
│  Output:                                                                    │
│    - success: boolean                                                       │
│    - data: string (enhanced message)                                        │
│    - error?: string                                                         │
└─────────────────────────────────────────────────────────────────────────────┘

Email Templates:

1. INTERVIEW_SCHEDULED
   Subject: "Interview Scheduled for {Job Title} at {Company}"

   Subject: "Update on your application for {Job Title}"

   - Personalized rejection message (from HR)

3. APPLICATION_SHORTLISTED
   Subject: "Great news about your {Job Title} application!"
   Content:
   - Congratulations on shortlisting
   - Next steps
   - Timeline expectations

================================================================================
                    SECURITY CONSIDERATIONS
================================================================================

1. Interview Link Security:
   - Include secure token in URL: /interviews/{id}?token={jwt}
   - Token validates: interviewId, candidateId, expiry
   - Rate limit joining attempts

2. Application Access:
   - Candidates can only view their own applications
   - HR can only view applications for their company's jobs

3. Action Authorization:
   - Only HR with appropriate role can reject/accept
   - Audit log all status changes
