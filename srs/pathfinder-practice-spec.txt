3. PROPOSED IMPROVEMENTS
================================================================================

3.1 Add Resources Tab (Before Quiz & Coding)
--------------------------------------------
When user adds a subgoal (e.g. "Learn beginner React"):
- Generate or fetch learning content:
  * Overview: What is React? Key Learns
  * Resources: Curated YouTube videos, docs links
  * Code examples: Snippets in read-only code editor
  * Best practices: Do's and Don'ts for the topic
- Display in a dedicated "Resources" or "Learn" tab
- Order: Resources first -> then Quiz -> then Coding
- Rationale: User should study before being tested

3.2 Subgoal-Level Resources
----------------------------
- Each subgoal gets its own resource set
- AI-generated or manually curated per topic
- When subgoal = "Beginner React": show React intro resources
- When subgoal = "Hooks": show Hooks-specific resources
- Quiz and Coding stay aligned to the subgoal

3.3 Studio Removal (For Now)
----------------------------
- Remove "Studio" button from Practice page header
- Remove studioId linkage from Pathfinder goal flow
- Keep Studio as separate product; integrate later if needed
- Practice page becomes the single learning hub for a goal

3.4 Code Examples - Read-Only
-----------------------------
- Use code editor component in read-only mode
- Show syntax-highlighted examples
- No edit/practice in this tab (that's the Coding tab)
- Future: "Try it" could open a sandbox (out of scope for now)

================================================================================
4. USER FLOW (DETAILED)
================================================================================

Step 1: User creates goal "Learn React - 30 days"
Step 2: AI generates plan with subgoals
Step 3: User adds/edits subgoals (e.g. "JSX basics", "Components", "Hooks")
Step 4: User opens Practice for a subgoal
Step 5: NEW - User sees Resources tab first
        - Reads overview
        - Watches recommended videos
        - Studies code examples
Step 6: User switches to Quiz tab
        - Attempts Learn questions
        - Gets feedback
Step 7: User switches to Coding tab
        - Solves coding challenges
        - Submits for verification
Step 8: When ready, user goes to Verification
        - Quiz section
        - Coding section
        - Mock interview
        - Project submission (if applicable)
